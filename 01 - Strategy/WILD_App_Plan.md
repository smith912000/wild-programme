## Executive Summary

The WILD Companion App is a guided practice tool that delivers the WILD Programme curriculum as a structured daily experience. It replaces passive PDF reading with an interactive, sensory-guided system — breathing tools, timed exercises, a dream journal, progress tracking, and community integration.

The app has two primary commercial roles:
1. **Tier unlock** — full app access is included in Tier 2 (£100) and Tier 3 (£200), creating a meaningful step-up from Tier 1
2. **Standalone upsell** — the app could be sold as a standalone product (£20–30 one-off or £5/mo subscription) to lead magnet users who don't want the full course

## Core App Sections

### 1. Breathing & Sound Studio
The highest-differentiation feature. No other lucid dreaming product offers this.

**Purpose:** Guide the user through the WBTB (Wake-Back-to-Bed) pre-sleep breathing protocol using visual and audio cues that prepare the nervous system for hypnagogia entry.

**Features:**
- **Animated breathing visualiser** — expanding/contracting orb or waveform, synchronized to breath phase
- **Three phases with distinct visual and audio cues:**
  - Inhale (4 sec): slow expansion + rising harmonic tone (e.g. 528 Hz sine wave, brightening orb)
  - Hold (4–7 sec): static pulsing + sustained harmonic drone
  - Exhale (4–8 sec): slow contraction + descending tone, orb dims to near-black
- **Preset protocols:**
  - Box breathing (4-4-4-4) — for general relaxation before WBTB
  - 4-7-8 (inhale 4, hold 7, exhale 8) — for rapid nervous system downregulation
  - Coherence breathing (5.5 sec in / 5.5 sec out) — for heart rate variability optimisation
  - Custom (user sets own durations)
- **Binaural audio layer** — optional theta or alpha binaural beats playing underneath breathing tones (6 Hz theta for sleep entry, 10 Hz alpha for WBTB alert-but-relaxed state)
- **Session timer** — 5, 10, or 20 minute presets
- **Haptic feedback** — optional phone vibration as a secondary breath cue (keeps eyes closed)

**Spiritual stream variant:**
- Same breathing tool, different visual: a moon phase animation replacing the orb, Tibetan singing bowl strikes replacing electronic tones
- Breath protocol framed as "pranayama" rather than biofeedback
- Can toggle between Bio and Spirit modes per user preference

### 2. Practice Library (Guided Exercises)

A structured library of guided audio + visual exercises, sorted by practice stage.

**Categories:**

**A. Entry Techniques (pre-sleep)**
- SSILD Cycles (Senses-Initiated Lucid Dreaming): timed visual, audio, and kinesthetic awareness cycles — 4 reps, 2 min each
- MILD Affirmation Builder: text prompt system to compose and audio-record a personal intention phrase
- Body Scan: 15-minute guided relaxation with progressive muscle release

**B. Hypnagogia Anchoring**
- Passive Awareness Practice: 10-minute session guiding the user to observe internal imagery without engaging it
- The Anchor Technique: instruction to maintain a single point of awareness (breath, or light point) as imagery develops
- Audio-hallucination identification: guidance on recognizing the shift from inner monologue to external-seeming voices

**C. Stabilisation (inside the dream)**
- First 60 seconds protocol: audio card with 4 grounding commands to use immediately upon achieving lucidity
- Rubbing hands / Spinning technique guides
- Demand clarity technique: "Increase clarity now" — when to use, how to use

**D. Integration (post-wake)**
- Dream recall ritual: guided 5-minute post-wake stillness with prompted journalling
- Emotion tagging: quick slide to rate emotional quality of dream
- Insight capture: what was unexpected? What was familiar?

**Library format:** Each exercise is a self-contained "session card" with:
- Duration (5–25 min)
- Difficulty (Beginner / Intermediate / Advanced)
- Best time of use (before WBTB / at WBTB / inside dream / upon waking)
- Stream tag (Bio / Spirit / Both)
- Audio guide (optional — user can run silent with text cues only)

### 3. Routine Board

A customisable daily practice board that structures the user's full WILD routine.

**Core components:**

**WBTB Alarm System:**
- Smart WBTB calculator — user enters bedtime, app calculates 5th/6th REM cycle entry (approx. 5.5–6 hours after sleep onset for most people)
- Alarm with gradual vibration (not jarring — designed to wake without full cortical activation)
- "Stay still" on-screen reminder when alarm fires
- 90-minute timer before sleep re-entry

**Pre-sleep Checklist (customisable, drag-and-drop):**
- Default items: No screens 30 min, Set journal ready, Set WBTB alarm, Set intention, Read protocol note, 10 min breathing exercise
- User can add, remove, reorder
- Check-off animation with subtle haptic feedback

**Morning Recall Ritual:**
- Triggered by morning alarm or manual open
- 5 minutes of guided stillness before journalling
- Voice memo option: record dream out loud before it fades
- Auto-timestamp everything

### 4. Dream Journal

A structured dream logging system with enough metadata to generate personal insights over time.

**Entry structure:**
- Date + time of waking
- WBTB used? (Y/N) — if yes, what time
- Lucid? (Y/N)
- If lucid: which technique triggered it? Duration estimate? Stability rating (1–5)
- Dream narrative (free text)
- Key emotions (multi-select tags: fear, curiosity, joy, confusion, awe, love, anxiety, peace)
- Key symbols (free text tags, builds a personal symbol library over time)
- Recurring elements (auto-detected from past entries)
- Clarity rating (1–5)
- Technique used (from preset list: WILD, WBTB, SSILD, MILD, other)

**Over time, the journal generates:**
- Most common emotions by sleep phase
- Recurring symbols + their trend
- Lucid success rate over time
- Average time to first lucid by technique used
- Best nights (day of week, moon phase as novelty, bedtime correlation)

## Tech Stack Recommendation

### Option A: Progressive Web App (PWA) — Recommended for MVP
**Estimated build cost:** £2,000–£6,000 with a freelancer or £0 with no-code tools
**Timeline:** 8–16 weeks

Why PWA first:
- Installable on iOS and Android without App Store approval or fees
- Works on desktop too (important for the biohacker audience who track data on PC)
- React or Vue frontend — much faster to build than native
- No 30% Apple/Google revenue cut on subscriptions
- Can progressively add native features (push notifications via web push, offline via service workers)

**Stack:**
- Frontend: React + Tailwind (consistent with landing page design system)
- Audio: Web Audio API (breathing tones, binaural beats, all generated in-browser — no audio file hosting needed)
- Animation: Framer Motion or CSS animations (breathing visualiser)
- Backend: Supabase (free tier — auth, database, file storage)
- Auth: Supabase Auth (magic link email — no password needed, very low friction)
- Payments: Integrated via Payhip/Whop webhook → unlock flag in Supabase
- Hosting: Vercel or Netlify (free tier)

### Option B: React Native — Native iOS + Android
**Estimated build cost:** £8,000–£20,000 (native dev required)
**Timeline:** 16–32 weeks

Better than PWA for:
- Haptic feedback on breathing exercises (true native haptics)
- Background audio playback (breathing timer continues when phone is locked)
- App Store presence (discoverability in App Store search)
- Push notifications (more reliable than web push)

Downside: 30% App Store / Play Store cut on subscriptions (unless users purchase via your web checkout and then verify on the app — which is allowed by Apple's rules but complex).

### Option C: No-Code (Bubble + Softr) — Fastest to Test
**Estimated build cost:** £0–£200/mo depending on tools
**Timeline:** 2–6 weeks

Tools:
- Bubble.io for the web app (breathing studio via custom HTML element, journal, dashboard)
- Softr for a simpler content-gated member portal
- Lottie animations for breathing visualiser
- Stripe integration for payments

Best use: Build the MVP to test whether users actually want the app before commissioning a native build.

**Recommendation: Start with Bubble.io or a React PWA. Validate demand at 500+ users before investing in native development.**
